JavaScript is everywhere

JavaScript is a programming language for a wide range of tasks:

  • Frontend (the main programming language)
  • Backend (Node.js, Deno, …)
  • Mobile (ReactNative, NativeScripts, Ionic, …)
  • IoT (JerryScript, QuickJS, …)
  • Desktop (NW.js, Electron, Neutralinojs, …)
It was originally created to add interactivity to web pages. But over time, it began to be used in completely different areas - on backend, on the Internet of things, to create native mobile applications, and other areas. Also, for some time now, it has become possible to use JavaScript to write native applications for desktops.

Why do we need desktop applications if there is an Internet?

Pros of Desktop apps

  • No internet required to run and use the app
  • No need to download anything to run
  • Have access to computer resources
  • Full control over the available features of the runtime
  • Backend independence

- No internet required to run and use the app

- At startup, you don’t need to wait for any data to be downloaded from the network

- Have access to computer resources

- Full control over the available features of the runtime. It does not depend on the web browser in which your code will run.

- If the server is undergoing maintenance or has problems, this will not affect the users in any way.

Electron

  • HTML, CSS and JavaScript are the basis
  • Share code between a desktop and a web applications
Electron allows you to create desktop applications using the tools and languages ​​that are familiar to many: HTML, CSS and JavaScript are the basis. This allows you to share code between a desktop application and a web applications.

Features

  • Web Technologies
  • Cross Platform
  • Native graphic user interface
  • Automatic software updates
  • Application installers
  • App store distribution
  • Crash reporting
Web Technologies
Electron embeds Chromium and Node.js.

Cross Platform.
Compatible with macOS, Windows, and Linux.

Native graphic user interface.
Electron can interact with operating system's interfaces by using special APIs.

Automatic software updates.
It can deliver a new version of applications with autoUpdater module.

Application installers.
It can generate installers for the main OSes.

App store distribution.
Electron has support stores of macOS, Windows and Linux.

Crash reporting.
Automatically collect JavaScript and native crash data with the crashReporter module.

Architecture

The architecture is very similar to modern browsers. Applications per electron are built around a single manager or "main" process that can run multiple "renderer" processes to render content.

The main process

  • Entry point for the entire application
  • Create and manage content windows
  • Manages the application life cycle
  • Interacting with the user's operating system
Serves as the entry point for the entire application.

It runs in a Node.js environment.

The main purpose is to create and manage content windows using the BrowserWindow module.

It also manages the application life cycle and contains an API for interacting with the user's operating system.

The renderer process

  • Display content
  • One window - one render process
Used to display content.
A separate render process is created for each open window.
The entry point for the process is an HTML file.

Practical part

Convert existing SPA into Desktop App

For the practical part, I will convert existing SPA from one of the tasks in RS School (Async race) into a desktop application.

Install Electron framework

The main requirement is Node.js installed. Recommended last LTS version.

  1. Create a folder and initialise npm in it

    
									mkdir electron-async-race      
									cd electron-async-race
									npm init

  2. Install Electron as dev-dependency

    
									npm install -D electron

  3. Add to package.json in the scripts section the line for start application:

    
									"scripts": {
										"start": "electron .",
									  },
First, let's install and create the basis of the Electron application.

Create the basis of the Electron application

  • Import the necessary modules:

    
									const { app, BrowserWindow } = require(‘electron’)

  • Create a function to load the HTML:

    
									const createWindow = () => {
										const win = new BrowserWindow({
											width: 600,
											height: 700
										})
										
										win.loadFile('index.html')
									}
									
									app.whenReady().then(() => {
										createWindow()
									})
Create the basis of the Electron application
In main.js file.
To display content in the application window, we need two Electron modules:
1) app module that is responsible for the application lifecycle
2) BrowserWindows is a module that creates and manages an application window.


The line win.loadFile('index.html') determines which HTML file will be loaded for display, which will be our entry point.

In line 11 we are waiting for the "ready" state for the app module. And only after that we can create our window with content.

Pros and cons of Electron

Let's talk about the pros and cons of Electron

Pros and Cons of Electron

Pros:

  • Ability to use familiar web technology stack to develop Desktop apps
Pros:
Ability to use familiar web technology stack to develop Desktop apps

Pros and Cons of Electron

Cons:

  • Requires a lot of resources
  • Not cross platform build
Cons:
Every Electron program is a full-fledged Chromium browser that uses CPU and memory. For example, our simple program takes up about 210 Mbytes of disk space and uses about 300 Mbytes of RAM.

Cross-platform to run, but not to build.
If the application has some native dependencies, then it can only be compiled on the target platform.

Conclusions

That's how easy we set up the environment and ported our existing project to Electron.
Of course, not all projects can be done so simply and may require further development.

Electron is a great framework for building desktop apps using web technologies. -- If you can write web applications, then you can write desktop applications too. -- But with great power comes great responsibility. -- It's best suited for building really big products where the overhead is justified.
